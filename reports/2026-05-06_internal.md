## What changed in the codebase

- `bin/apply_curation.R` — new. Reads `reports/<date>_curation.xlsx`, applies sample > study > heuristic precedence, writes `results_full_v2/metadata/cell_line_flags_curated.tsv` with a per-row `source` column.
- `bin/sweep.R` — prefers `cell_line_flags_curated.tsv` if present; falls back to `cell_line_flags.tsv`. No other behavioural changes.
- `bin/cell_line_patterns.py` — unchanged. The heuristic patterns are not retroactively updated to match what curation revealed; pattern improvements are a separate consideration (notably: pLVX vector family for the SRP539392 case, ASO treatment markers for ERP149729).

## Why: §2 caveat from the 2026-05-05 internal report

The earlier report noted "primary cells aren't cell lines but transduced/edited cells get their own flag." The curation surfaced two cases the heuristic didn't anticipate:

1. **A study can be heuristic-engineered but actually cell-line**: SRP514210 (LV-STK38 in A375 melanoma cells) and SRP545498 (siRNA in HaCaT) — the heuristic matched the construct prefix (`LV-`, `si-`) before checking the underlying line. Not a heuristic error per se — both interpretations are defensible — but study-level curation lets us prefer the line-of-record interpretation.
2. **A clinical-looking primary culture can be engineered**: SRP539392 (`FB_PLVX_80_Rep1`) names the lentiviral vector in the title but uses no shRNA / siRNA / sgRNA convention. Adding `pLV[A-Z]\w+` to `ENGINEERED_PATTERNS` would catch this prospectively.

Both classes are now reproducible via the curation workbook flow rather than ad-hoc pattern tweaking, which is the correct trade-off.

## Open questions / next steps

1. **Per-sample curation for ERR10293422 / ERR10293427.** Confirm whether the 2 surviving HPV16-in-skin hits are real or technical (cross-mapping, library carry-over). At depths < 3, neither is trivially separable.
2. **Pattern updates.** Decide whether to backfill `pLVX` / `pLKO` / `pBABE` / `pCDH` (Addgene retro/lentiviral backbone family) into `ENGINEERED_PATTERNS`. Evidence-driven would be: only add patterns that map to ≥ 2 confirmed misses in production runs.
3. **Re-curate after next chunk.** Studies fully done: 11/457; partially done: 374; pending: 72. New typed studies will appear as more chunks complete; rerun `bin/build_curation_workbook.R` periodically.
